# Google Sheets 구조 설계 문서

## 시트 탭 목록

### 1. Topics_Pool (주제 풀)
무한 생성 가능한 주제 창고

| 컬럼 | 설명 | 예시 |
|------|------|------|
| A: 카테고리 | 주제 분류 | 경제·생활, IT·디지털, 직장인, 생활·기타 |
| B: 주제 키워드 | 핵심 키워드 | ETF, 라우터, 4대보험, 치약 |
| C: 한 줄 설명 | 주제에 대한 간단 설명 | 주식 투자 시 분산투자 방법 |
| D: 사용 여부 | unused / planned / used | unused |

### 2. Weekly_Trend (주간 트렌드)
이번 주 검색 트렌드 키워드

| 컬럼 | 설명 | 예시 |
|------|------|------|
| A: 날짜 | 기록 날짜 | 2025-01-20 |
| B: 키워드 | 트렌드 키워드 | 전기요금 |
| C: 소스 | 출처 (유튜브/네이버/구글) | 네이버 |
| D: 관련도 | 높음/보통/낮음 | 높음 |
| E: 메모 | 참고사항 | 가격 인상 이슈 |

### 3. Weekly_Plan (주간 계획)
이번 주 업로드 예정 2개

| 컬럼 | 설명 | 예시 |
|------|------|------|
| A: 주차 | ISO 주차 형식 | 2025-W21 |
| B: 업로드 예정일1 | 첫 번째 영상 날짜 | 2025-01-22 |
| C: 업로드 예정일2 | 두 번째 영상 날짜 | 2025-01-25 |
| D: 주제1 | 첫 번째 주제 키워드 | ETF |
| E: 주제2 | 두 번째 주제 키워드 | 라우터 |
| F: 트렌드 키워드 | 참고용 트렌드 | 전기요금 |

### 4. Shorts_Content (숏츠 콘텐츠)
실제 생성된 콘텐츠 정보

| 컬럼 | 설명 | 예시 |
|------|------|------|
| A: ID | 자동 번호 | 1 |
| B: 업로드 주차 | 주차 정보 | 2025-W21 |
| C: 업로드 목표 날짜 | 목표 업로드일 | 2025-01-22 |
| D: 상태 | 작성중/촬영중/편집중/업로드완료 | 작성중 |
| E: 주제 키워드 | 주제 | ETF |
| F: 최종 제목 | 유튜브 제목 | 이거 진짜 모르는 사람 많습니다 |
| G: YouTube 설명란 | 설명 텍스트 | ETF에 대해 3분만에 알아보자... |
| H: 해시태그 | 해시태그 리스트 | #ETF #투자 #주식 #재테크 |
| I: 대본(스크립트) | 60초 스크립트 | Hook: 이거 진짜 모르는... |
| J: 한 줄 훅 | 첫 문장 | 이거, 진짜 모르는 사람 많습니다. |
| K: 참고 트렌드 키워드 | 관련 트렌드 | 전기요금 |
| L: 참고 영상 링크 | References ID 또는 URL | ref-1, https://... |
| M: 메모 | 추가 메모 | 촬영 시 주의사항 |

### 5. References (참고 자료)
레퍼런스 DB

| 컬럼 | 설명 | 예시 |
|------|------|------|
| A: 레퍼런스 ID | 고유 ID | ref-1 |
| B: 타입 | 영상/문서/기사/블로그 | 영상 |
| C: 링크(URL) | URL | https://youtube.com/... |
| D: 관련 주제 키워드 | 태그 | ETF, 투자 |
| E: 메모 | 좋았던 포인트 | 간단명료한 설명 |
| F: 사용 여부 | 사용함/참고만 | 참고만 |

## 시트 생성 순서

1. Google Sheets 새 스프레드시트 생성
2. 기본 시트 이름 변경: Sheet1 → Topics_Pool
3. 위 컬럼 헤더 작성
4. 새 탭 추가: Weekly_Trend, Weekly_Plan, Shorts_Content, References
5. 각 탭에 헤더 작성

## 참고사항

- 첫 행은 항상 헤더
- 모든 시트는 첫 행부터 데이터 시작
- ID는 자동 생성 또는 수동 입력
